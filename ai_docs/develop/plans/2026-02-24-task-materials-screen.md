# План: Отдельный экран управления материалами задачи

**Создан:** 2026-02-24  
**Оркестрация:** orch-2026-02-24-14-00-task-materials  
**Статус:** 🔵 Планирование  
**Приоритет:** Средний

## Цель

Создать отдельный полноэкранный экран для управления материалами задачи графика (аналогично SelectActTemplate), заменив встроенный компонент TaskMaterialsEditor в диалоге редактирования задачи на кнопку перехода.

## Контекст

### Текущая реализация
- В `Schedule.tsx` есть компонент `TaskMaterialsEditor` внутри диалога редактирования задачи (строки 1283-1298)
- Материалы задачи хранятся в таблице `task_materials` (связь с `project_materials`)
- Есть хуки `useTaskMaterials` и `useReplaceTaskMaterials` для работы с API
- Компонент `TaskMaterialsEditor` позволяет добавлять/удалять материалы, указывать batch/quality document/note

### Проблема
- Редактирование материалов в диалоге неудобно на мобильных устройствах
- Ограниченное пространство для списка материалов
- Нет возможности изменить порядок материалов (drag-and-drop)

### Решение
- Создать отдельную страницу `SelectTaskMaterials.tsx` по аналогии с `SelectActTemplate.tsx`
- Использовать навигацию через `wouter` с передачей `taskId` через `history.state`
- Сохранять изменения при возврате назад
- Добавить возможность изменения порядка материалов

## Архитектурные решения

### Навигация
- Использовать паттерн из `SelectActTemplate.tsx`:
  - Передача данных через `window.history.state`
  - Возврат через `window.history.back()`
  - Обновление данных через invalidate query

### Сохранение данных
- **Вариант 1 (рекомендуемый):** Автосохранение при изменении (debounce 500ms)
  - Плюсы: Не нужно думать о сохранении, UX как в современных приложениях
  - Минусы: Больше запросов к API
  
- **Вариант 2:** Кнопка "Сохранить" в footer
  - Плюсы: Меньше запросов, явное действие
  - Минусы: Можно забыть сохранить

**Решение:** Использовать вариант 1 (автосохранение) для лучшего UX.

### Изменение порядка материалов
- **MVP:** Простые кнопки "Вверх/Вниз"
- **Будущее:** Drag-and-drop с библиотекой `@dnd-kit/core`

### Валидация
- Проверять наличие `taskId` при загрузке страницы
- Если `taskId` отсутствует → показать ошибку и кнопку "Вернуться к графику"
- Проверять уникальность материалов (нельзя добавить один материал дважды)

## Риски и ограничения

### Риски
1. **Потеря данных при навигации:**
   - Митигация: Автосохранение при каждом изменении
   
2. **Производительность при большом количестве материалов:**
   - Митигация: Виртуализация списка (если > 100 элементов)
   
3. **Конфликты при одновременном редактировании:**
   - Митигация: Оптимистичные обновления + обработка ошибок

### Ограничения
- Нет drag-and-drop в MVP (только кнопки вверх/вниз)
- Нет поддержки batch selection (выбор нескольких материалов сразу)
- Нет истории изменений (undo/redo)
