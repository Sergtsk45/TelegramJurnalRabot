# Plan: Исправление диалога «Редактирование задачи» на мобильных

**Создан:** 2026-02-25  
**Цель:** Исправить UX диалога редактирования задачи на вкладке «График работ» (Schedule.tsx) для мобильных устройств.

## Проблемы
1. Диалог «ездит» влево-вправо при свайпе на телефоне (нестабильность)
2. Клавиатура появляется сразу при открытии, вместо этого должна — только после клика по полю (номер акта или длительность)

## Подзадачи

1. **[UI-001]** ✅ Отключить автофокус при открытии — передать `onOpenAutoFocus={(e) => e.preventDefault()}` в `DialogContent` диалога редактирования задачи (Schedule.tsx, ~стр. 1139). Radix Content поддерживает этот проп — он отменяет автофокус на первый инпут, клавиатура не будет появляться до явного тапа по полю.

2. **[UI-002]** ✅ Исключить горизонтальный дрейф при свайпе — добавлены `touch-pan-y`, `overscroll-contain`, `overflow-x-hidden` на DialogContent и scroll-контейнер; `touch-none` на overlay в dialog.tsx.

3. **[UI-003]** (опционально) Вынести общие мобильные стили в `dialog.tsx` — частично: `touch-none` добавлен на overlay; остальные стили оставлены только для диалога редактирования задачи.

## Файлы
- `client/src/pages/Schedule.tsx` (диалог ~1133–1312)
- `client/src/components/ui/dialog.tsx` (базовый компонент, при UI-003)

## Зависимости
- UI-001, UI-002 — независимы, можно выполнять параллельно
- UI-003 — после проверки эффективности UI-002
