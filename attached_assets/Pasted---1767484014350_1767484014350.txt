Цель сервиса

Автоматизация создания исполнительной документации (АОСР, акты скрытых работ и т.п.) на основе голосовых/текстовых сообщений через Telegram miniapp, с дальнейшим хранением данных в SQLite и выгрузкой в Excel при необходимости. Все сообщения должны быть нормализованы и привязаны к заранее загруженной ведомости объемов работ (ВОИР).

🧩 Архитектура и компоненты
Интерфейс:

Telegram MiniApp (инлайн-режим, кнопки, инпуты).

Голосовой и текстовый ввод от пользователя.

Хранилище:

SQLite (основная БД с таблицами: работы, участки, сотрудники, акты).

Excel — только для вывода (генерация финального документа).

Подключаемые ресурсы:

Ведомость объемов работ (загружается в начале проекта — в Excel или CSV).

Сертификаты, паспорта, тех. доки — загружаются пользователем по запросу, хранятся с привязкой к акту.

📋 Предполагаемая структура данных
Таблица «сообщения» (сырой ввод)

id

user_id

datetime (дата и время записи)

message_raw (оригинальное сообщение)

message_normalized (нормализованная версия)

Таблица «работы»

id

код_работы

описание

единица_измерения

объем_по_ведомости

нормализованные_синонимы (для сопоставления с вводом)

Таблица «акты»

id

дата_начала

дата_окончания

участок

работы (json: виды работ, объемы, исполнители)

прикрепленные_документы

🔁 Бизнес-логика

Пользователь вводит голосовое или текстовое сообщение — «вчера забетонировали плиту на первом этаже».

STT — преобразует речь в текст.

LLM + нормализация — определяет тип работ, дату, объем (если есть), участок, рабочую зону, исполнитель, и приводит к терминам из ВОИР.

Сопоставление с ведомостью — проверяется наличие работы в справочнике, подбирается код работы.

Запись в SQLite — нормализованные данные заносятся в базу.

По команде “собери акт за…” — система агрегирует работы, формирует таблицу акта.

Запрос на прикрепление документов — отправляется сообщение с кнопками или ссылкой для загрузки.

Генерация акта в PDF или Excel — по шаблону.

📎 Требуются уточнения

Чтобы составить финальное ТЗ, нужно прояснить:

В каком виде предоставляется ВОИР (Excel, CSV)? Какая структура?

Будет ли акт формироваться автоматически по дням или только по команде?

Какие поля обязательны для голосового ввода? (Например: работа, дата, объем, место, ФИО?)

Где пользователь будет прикреплять документы (Telegram, веб-ссылка)?

Какие роли пользователей: только мастер или еще прораб, технадзор?

Итоговый формат акта — PDF, Word, Excel?
